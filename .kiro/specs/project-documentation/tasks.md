# Implementation Plan: Проєктна документація

## Overview

Створення єдиного Markdown-документа `docs/PROJECT_OVERVIEW.md` з повним описом проєкту рейтингової системи українського «Що? Де? Коли?». Документ охоплює продуктову цінність, архітектуру, технологічний стек та конвенції розробки.

    - Додати заголовок: `# Огляд проєкту: Рейтинг «Що? Де? Коли?»`
    - Додати блок метаданих із версією документа та датою останнього оновлення
    - Створити зміст (Table of Contents) із Markdown-посиланнями (anchor links) на всі основні секції та підсекції
    - Документ пишеться українською мовою

    - Додати Mermaid-діаграму взаємозв'язків модулів Common та Classic
    - Описати модуль Common (загальний код: автентифікація, базові сутності, валідація, хелпери, спільні DTO)

  - [x] 3.2 Створити підсекції «Шарова архітектура», «Потік даних» та «Кешування» з Mermaid-діаграмою
    - Додати Mermaid-діаграму шарової архітектури (Controller → Service → Repository → Entity → DTO)
    - Описати шарову архітектуру кожного модуля (Controller, Service, Repository, Entity, DTO, Mapping)
    - Пояснити потік даних: Controller приймає DTO → Service обробляє → Repository працює з БД → Entity маппиться на DTO
    - Описати роль кешування (Redis із tag-based інвалідацією, кешування DTO, каскадна інвалідація)

## Notes

- Артефактом є статичний Markdown-файл, тому property-based тести не застосовні
- Кожне завдання посилається на конкретні вимоги для трасування
- Зміст (TOC) створюється вручну з Markdown anchor links
- Mermaid-діаграми мають рендеритися на GitHub без зовнішніх залежностей
- Секція конвенцій не дублює steering-файл, а подає стислий огляд із посиланням
